- Monitoring
	- Tools to check metrics such as CPU usage, and set alerts.

- AWS CloudWatch
	- Provides metrics for every service in AWS
		- EC2: CPU utilization
		- S3 bucket: NumberOfObjects, BucketSizeBytes
		- Create custom metrics
	- Can create alarms to have triggered notifications.
		- States: OK, INSUFFICIENT_DATA, ALARM
	- Centralizes logs from all services into one place.
		- Can view, search, filter, archive, and query them.
		- Two types: Logs Standard and Logs Infrequent Access.
		- Spins up log agents that live on EC2 instances or on prem servers to push log files when needed.

- Amazon EventBridge
	- Serverless scheduler, schedules based on a response.
		- Cron jobs (based on time) or recurring timer.
		- Or have rules that reacts to certain events (like an alarm but it performs an action).

- AWS CloudTrail
	- Provides governance, compliance, and audit for your AWS account, default-ly activated.
		- Trails which capture a record of activities.

- AWS X-Ray
	- Collects data about requests that your application serves and provides tools to simplify log analysis and debugging.
		- Dives deep into services, showing access times, response rates, etc.

- AWS Health
	- The health dashboard shows resource performance and the availability of all services.
